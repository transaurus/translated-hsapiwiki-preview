# Hopscript

## 基本目標

Hopscriptの最も重要な目標は以下の通りです:

- あらゆるHopscotchコードをテキスト形式で表現すること
- 通常のHopscotchからの移行をシンプルに理解できること
- 既存のHopscotchコードとの双方向の簡単なインターフェースを提供すること

## 二次的目標

これらは依然として非常に重要ですが、上記の目標ほどではありません。

- 1対1のデホップスコッチ化 - 通常のHopscotchコードをHopscriptに変換し、内部IDなどの特定の要素を無視して同一のコードに再変換できること
- ユーザー定義可能なブロック - モッド作成に有用
- 名前のシャドウイングへの適切な対応 - `and`というローカル変数がある場合、`and`ブロックと明確に区別でき、変数名として適切でなくてもコンパイラが処理できること
- 容易な拡張性 - この投稿の後半でいくつかのアイデアを提案予定

## 実装予定リスト

Hopscriptの完成版に向けて必要な作業のリストです。同時に実施可能な項目は単一の番号見出しにまとめています

#### ステップ1

- Hopscotch化に必要なhsapiの初版を作成
- パーサーの実装
- hsapiを必要としない拡張機能セクションの一部を実装

#### ステップ2

- Hopscotch化ツールの作成

#### ステップ3

- Hopscriptを使用したプロジェクト作成と構文改善
- デホップスコッチ化ツールの作成
- 拡張機能インタープリターの作成

#### ステップ4

- 基本Hopscriptで拡張機能をより活用開始（例: スコープのハードコード定義の削除）
